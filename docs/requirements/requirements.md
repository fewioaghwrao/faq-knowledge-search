## 1. テーマ・対象ナレッジ領域

本システムが対象とするナレッジ領域は、**社内業務アプリの操作手順・障害対応ナレッジ** とする。

対象とするFAQ例は以下である。

- ログインできない場合の対応
- 権限エラーが表示された場合の確認手順
- CSV取込エラーへの対応
- PDF出力失敗時の確認手順
- APIエラー・タイムアウト時の一次対応
- 月次処理・締め処理の操作手順
- メール通知が届かない場合の確認手順
- 業務アプリの登録・更新・検索操作に関する問い合わせ

本システムでは、これらのFAQ・手順書・障害対応メモを登録し、通常検索およびAI検索から必要な情報を確認できることを目的とする。

## 2. プロジェクト概要

### 2.1 背景

社内業務では、操作手順や障害対応方法が担当者の経験、チャット履歴、個別メモ、手順書などに分散しやすい。

その結果、以下のような課題が発生する。

| 課題 | 内容 |
|---|---|
| 情報が探しにくい | FAQや手順書が分散し、必要な情報にたどり着くまで時間がかかる |
| 属人化しやすい | 対応方法を知っている担当者に確認が集中する |
| 同じ問い合わせが繰り返される | よくある質問がナレッジ化されず、対応コストが増える |
| 新人・引き継ぎ時に困る | 過去対応や業務手順を体系的に確認しづらい |
| AI回答の根拠が不明になりやすい | AIだけの回答では、どのFAQを根拠にしたか確認しづらい |

---

### 2.2 目的

本システムの目的は、社内FAQ・業務ナレッジを一元管理し、担当者が必要な情報をすばやく検索・確認できるようにすることである。

また、AI検索では、FAQを根拠にした回答を生成し、回答の参照元FAQを表示することで、AI回答の説明可能性を高める。

---

### 2.3 システム概要

本システムは、以下の機能を持つWebアプリケーションである。

- FAQ一覧表示
- FAQ詳細表示
- キーワード検索
- FAQ登録・編集・削除
- FAQ公開 / 非公開管理
- 管理者ログイン
- ユーザー管理
- AI FAQ検索
- AI回答表示
- 参照元FAQ表示
- AI検索履歴管理
- AI回答フィードバック
- テスト
- デプロイ

## 6. 前提条件・制約

| 項目 | 内容 |
|---|---|
| データ | 実案件情報・機密情報は含めず、ポートフォリオ用のダミーデータを使用する |
| 認証 | 管理者向け機能はJWT認証を利用する |
| AI | OpenAI APIを利用し、FAQコンテキストに基づく回答を生成する |
| AI安全性 | FAQにない内容は断言しないようにプロンプトで制御する |
| DB | MySQLを利用する |
| ブラウザ | Chrome / Edge の最新版を想定する |
| デプロイ | FrontendはAzure Static Web Apps、BackendはHerokuに配置する |
| APIキー | User Secrets またはホスティング環境の環境変数で管理する |
| 個人情報 | 実在の個人情報・顧客情報・認証情報は登録しない |

## 9. 機能詳細

### 9.1 FAQ一覧・検索

| 項目 | 内容 |
|---|---|
| 概要 | 登録済みFAQを一覧表示し、キーワードで絞り込む |
| 利用者 | 一般利用者、管理者 |
| 検索対象 | タイトル、本文、カテゴリ、タグ |
| 表示項目 | タイトル、本文抜粋、カテゴリ、タグ、閲覧数、更新日時 |
| 条件 | 一般利用者には公開中FAQのみ表示する |
| ソート | 関連度順または更新日時順 |
| 補足 | 検索キーワードに一致する箇所を強調表示する |

### 9.2 FAQ詳細表示

| 項目 | 内容 |
|---|---|
| 概要 | 選択したFAQの詳細情報を表示する |
| 表示項目 | タイトル、本文、カテゴリ、タグ、閲覧数、作成日時、更新日時 |
| 条件 | 非公開FAQは管理者のみ閲覧可能 |
| 補足 | 詳細表示時に閲覧数を加算する |

### 9.3 FAQ登録・編集・削除

| 項目 | 内容 |
|---|---|
| 概要 | 管理者がFAQを登録・編集・削除する |
| 利用者 | 管理者 |
| 入力項目 | タイトル、本文、カテゴリ、タグ、公開状態 |
| 削除方式 | 論理削除 |
| バリデーション | タイトル必須、本文必須、カテゴリ必須 |

### 9.4 AI FAQ検索

| 項目 | 内容 |
|---|---|
| 概要 | ユーザーの質問に対し、関連FAQを検索してAI回答を生成する |
| 利用者 | 一般利用者、管理者 |
| 入力項目 | 質問文 |
| 処理 | 質問文からキーワードを抽出し、関連FAQ上位5件をAIコンテキストとして利用する |
| 出力 | AI回答、注意文、参照元FAQ |
| FAQ0件時 | AI回答を生成せず、該当FAQなしのメッセージを表示する |

### 9.5 参照元FAQ表示

| 項目 | 内容 |
|---|---|
| 概要 | AI回答の根拠として利用したFAQを表示する |
| 表示項目 | FAQタイトル、FAQ詳細リンク |
| 目的 | AI回答の根拠を確認できるようにする |
| 注意文 | 「詳細は参照元FAQをご確認ください。」を表示する |

### 9.6 AI検索履歴管理

| 項目 | 内容 |
|---|---|
| 概要 | AI検索の実行履歴を保存し、管理画面で確認する |
| 保存項目 | 質問文、検索キーワード、AI回答、参照元FAQ、成否、エラー内容、フィードバック、実行日時 |
| 一覧 | 管理者が履歴を一覧表示する |
| 詳細 | 個別履歴の質問、回答、参照元FAQを表示する |
| フィードバック | 役に立った / 役に立たなかったを記録する |

### 9.7 ユーザー管理

| 項目 | 内容 |
|---|---|
| 概要 | 管理者がユーザー情報を確認・管理する |
| 表示項目 | 表示名、メールアドレス、ロール、有効状態、作成日時 |
| 操作 | 有効 / 無効切り替え、ロール管理 |
| 制約 | 自分自身の無効化は不可 |

|---|---|
| FAQ検索 | 通常利用時に数秒以内で検索結果を表示する |
| AI検索 | AI API呼び出しを含むため、タイムアウトを設定する |
| 一覧表示 | ページネーションを利用し、必要な件数のみ取得する |
| AIコンテキスト | AIに渡すFAQ件数は上位5件程度に制限する |

### 13.2 セキュリティ

| 項目 | 要件 |
|---|---|
| 認証 | JWT認証を利用する |
| 管理機能 | 管理者のみ利用可能とする |
| APIキー | User Secrets またはホスティング環境の環境変数で管理する |
| CORS | 許可されたフロントエンドURLのみ許可する |
| SQLインジェクション対策 | EF Coreを利用する |
| XSS対策 | 画面表示時に適切にエスケープする |
| 機密情報 | FAQ・AI回答・ログに機密情報を含めない |
| 無効ユーザー | `IsActive = false` のユーザーはログイン不可とする |

### 13.3 AI安全性

| 項目 | 要件 |
|---|---|
| 根拠制御 | FAQコンテキストに基づく回答に限定する |
| 断言抑制 | FAQにない内容を断言しない |
| 参照元表示 | AI回答には参照元FAQを表示する |
| 個人情報対策 | 個人情報・認証情報を出力しない |
| 注意文 | 「詳細は参照元FAQをご確認ください。」を表示する |

### 13.4 保守性

| 項目 | 要件 |
|---|---|
| テスト | バックエンド・フロントエンドの主要テストを実装する |
| ログ | エラー内容を確認できるようにする |
| 設定 | 環境変数・User Secretsで環境差分を管理する |
| ドキュメント | README、基本設計書、詳細設計書、ER図、画面遷移図を整備する |
