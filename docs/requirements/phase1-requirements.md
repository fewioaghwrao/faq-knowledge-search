## 1. プロジェクト概要

### 1.1 背景・目的

本システムは、社内業務担当者・カスタマーサポート担当者・システム運用担当者が、FAQ・手順書・障害対応メモを検索し、業務上の疑問やエラー対応方法を素早く確認するためのナレッジ検索システムである。

請求処理、CSV取込、ログイン障害、API障害、PDF出力など、業務システムで発生しやすい操作手順やエラー対応を効率よく検索・参照できる社内FAQ管理アプリをフェーズ1として構築する。フェーズ2以降でAI要約・RAG検索機能を追加予定。

### 1.2 想定利用者

本システムの主な利用者は以下のとおりである。

#### 一般利用者

FAQや手順書を検索・閲覧するユーザー。業務中に発生した疑問、操作手順、エラー対応方法をキーワードやタグから検索する。

| ユーザー | 主な使い方 |
|---|---|
| 経理担当者 | 請求処理・CSV取込・月次締めの手順を確認する |
| カスタマーサポート | 顧客からの問い合わせに対してFAQを検索する |
| 社内ヘルプデスク | ログイン障害・権限エラー・PDF出力不具合の対応手順を探す |
| 業務システム運用担当 | API障害・取込エラー・エラーパターンの過去対応を確認する |
| 新人・引き継ぎ担当者 | 業務手順や障害対応メモを学ぶ |

#### ナレッジ管理者

FAQ・カテゴリ・タグを登録・編集・削除するユーザー。検索履歴や利用状況を確認し、不足しているFAQの追加や内容改善を行う。

#### システム管理者

ユーザー管理・権限管理・公開状態の管理を行うユーザー。フェーズ1では最低限の管理機能のみ実装する。

### 1.3 スコープ

| 対応範囲（In Scope） | 対応外（Out of Scope） |
|---|---|
| FAQ一覧・詳細表示 | AI要約・RAG検索（フェーズ2以降） |
| FAQ登録・編集・削除 | 外部システム連携（Slack・Teams通知等） |
| カテゴリ管理 | モバイルアプリ対応 |
| タグ管理 | 多言語対応 |
| キーワード検索・タグ検索 | 高度な権限・組織管理 |
| 検索履歴の記録・表示 | — |
| 公開/非公開管理 | — |

| 項目 | 内容 |
|---|---|
| 技術スタック | ASP.NET Core Web API / Next.js / Azure App Service / SQL Server（本番）・LocalDB（開発環境） |
| ランタイム | Node.js 22 / .NET 8 |
| 対応ブラウザ | Chrome 最新版 / Edge 最新版 |
| 認証 | フェーズ1はJWT認証を実装。ユーザー種別は一般利用者・管理者の2種類。ユーザー新規登録・メール認証・パスワード再設定・二要素認証・外部ID連携はフェーズ1対象外 |

| データ | FAQデータはダミーデータで初期登録。実案件情報は含まない |
| その他制約 | — |

### 1.5 用語定義

| 用語 | 定義 |
|---|---|
| FAQ | よくある質問と回答のセット。タイトル・本文・カテゴリ・タグで構成される |
| カテゴリ | FAQを業務領域で分類するグループ（例：請求処理、CSV取込） |
| タグ | FAQに付与するキーワードラベル。複数付与可能 |
| 検索履歴 | ユーザーが入力したキーワードの履歴。FAQ改善のために参照する |
| 公開/非公開 | FAQの表示状態。非公開のFAQは一般利用者に表示されない |
| ナレッジ管理者 | FAQの登録・編集・管理を担当するロール |

## 2. 機能要件

### 2.1 機能一覧

| ID | 機能名 | 概要 | 優先度 |
|---|---|---|---|
| F-001 | FAQ一覧表示 | 登録済みFAQをカテゴリ・タグ・キーワードで絞り込み一覧表示する | Must |
| F-002 | FAQ詳細表示 | 選択したFAQのタイトル・本文・カテゴリ・タグを表示する | Must |
| F-003 | FAQ登録 | 管理者がFAQを新規登録する（タイトル・本文・カテゴリ・タグ・公開状態） | Must |
| F-004 | FAQ編集 | 管理者が既存FAQの内容を編集する | Must |
| F-005 | FAQ削除 | 管理者がFAQを削除する（論理削除） | Must |
| F-006 | キーワード検索 | タイトル・本文に対する部分一致検索を行う | Must |
| F-007 | タグ検索 | 指定タグが付与されているFAQを絞り込む | Must |
| F-008 | カテゴリ管理 | カテゴリの追加・編集・削除を行う | Must |
| F-009 | タグ管理 | タグの追加・編集・削除を行う | Should |
| F-010 | 検索履歴 | ユーザーの検索キーワード履歴を記録・表示する | Should |
| F-011 | 公開/非公開管理 | FAQの公開状態を切り替える | Should |

### 2.2 機能詳細

#### F-001: FAQ一覧表示

| 項目 | 内容 |
|---|---|
| 概要 | 登録済みのFAQをカテゴリ・タグ・キーワードで絞り込んで一覧表示する |
| アクター | 一般利用者・ナレッジ管理者 |
| 事前条件 | FAQが1件以上登録されていること |
| 基本フロー | ①一覧画面を開く → ②FAQがページネーション付きで表示される → ③カテゴリ/タグ/キーワードで絞り込む → ④結果が更新される |
| 代替フロー | 検索結果が0件の場合は「該当するFAQが見つかりません」を表示する |
| ビジネスルール | 非公開のFAQは一般利用者には表示しない。管理者は全件表示可能 |

#### F-002: FAQ詳細表示

| 項目 | 内容 |
|---|---|
| 概要 | 一覧から選択したFAQのタイトル・本文・カテゴリ・タグ・更新日時を表示する |
| アクター | 一般利用者・ナレッジ管理者 |
| 事前条件 | 対象FAQが存在すること・公開状態であること（一般利用者の場合） |
| 基本フロー | ①一覧でFAQを選択 → ②詳細画面に遷移 → ③本文・タグ・カテゴリを確認できる |
| 代替フロー | 存在しないIDにアクセスした場合は404ページを表示する |
| ビジネスルール | 本文はMarkdown形式で登録・表示することを推奨する |

#### F-003: FAQ登録

| 項目 | 内容 |
|---|---|
| 概要 | ナレッジ管理者がFAQを新規登録する |
| アクター | ナレッジ管理者 |
| 事前条件 | カテゴリが1件以上登録されていること |
| 基本フロー | ①登録フォームを開く → ②タイトル・本文・カテゴリ・タグ・公開状態を入力 → ③登録ボタンを押す → ④一覧に反映される |
| 代替フロー | バリデーションエラー時はエラーメッセージを入力欄の下に表示し、登録は行わない |
| ビジネスルール | タイトルは必須・最大100文字。本文は必須・最大10,000文字。カテゴリは必須 |

#### F-006: キーワード検索

| 項目 | 内容 |
|---|---|
| 概要 | 入力したキーワードでFAQのタイトル・本文を部分一致検索する |
| アクター | 一般利用者・ナレッジ管理者 |
| 事前条件 | なし |
| 基本フロー | ①検索バーにキーワードを入力 → ②検索実行（リアルタイムまたはボタン押下） → ③マッチしたFAQが一覧表示される → ④検索キーワードが履歴に記録される |
| 代替フロー | スペース区切りでAND検索とする。0件の場合は該当なしメッセージを表示 |
| ビジネスルール | 検索は公開中のFAQのみ対象（管理者は全件対象）。大文字・小文字は区別しない |

#### F-010: 検索履歴

| 項目 | 内容 |
|---|---|
| 概要 | ユーザーが検索したキーワードをブラウザセッション単位で記録・表示する |
| アクター | 一般利用者・ナレッジ管理者 |
| 事前条件 | 過去にキーワード検索が実行されていること |
| 基本フロー | ①検索バーをクリック → ②直近の検索履歴が最大10件表示される → ③履歴を選択すると再検索される |
| 代替フロー | 履歴が0件の場合は「検索履歴はありません」と表示する |
| ビジネスルール | 同一キーワードは重複して記録しない（最新日時に更新）。履歴はSearchHistoriesテーブルにDB保存する（フロント表示用はローカルキャッシュ可） |

## 4. API設計

### 4.1 API一覧

| ID | メソッド | エンドポイント | 認証 | 概要 |
|---|---|---|---|---|
| A-001 | GET | /api/v1/faqs | 任意（JWT） | FAQ一覧取得（キーワード・カテゴリ・タグ絞り込み対応） |
| A-002 | GET | /api/v1/faqs/{id} | 任意（JWT） | FAQ詳細取得 |
| A-003 | POST | /api/v1/faqs | 要（JWT・管理者） | FAQ新規登録 |
| A-004 | PUT | /api/v1/faqs/{id} | 要（JWT・管理者） | FAQ更新 |
| A-005 | DELETE | /api/v1/faqs/{id} | 要（JWT・管理者） | FAQ削除（論理削除） |
| A-006 | GET | /api/v1/categories | 任意（JWT） | カテゴリ一覧取得 |
| A-007 | POST | /api/v1/categories | 要（JWT・管理者） | カテゴリ登録 |
| A-008 | PUT | /api/v1/categories/{id} | 要（JWT・管理者） | カテゴリ更新 |
| A-009 | DELETE | /api/v1/categories/{id} | 要（JWT・管理者） | カテゴリ削除 |
| A-010 | GET | /api/v1/tags | 任意（JWT） | タグ一覧取得 |
| A-011 | POST | /api/v1/tags | 要（JWT・管理者） | タグ登録 |
| A-012 | PUT | /api/v1/tags/{id} | 要（JWT・管理者） | タグ更新 |
| A-013 | DELETE | /api/v1/tags/{id} | 要（JWT・管理者） | タグ削除 |

### 4.2 API詳細

#### A-001: FAQ一覧取得

| 項目 | 内容 |
|---|---|
| エンドポイント | GET /api/v1/faqs |
| クエリパラメータ | keyword: string（部分一致）、categoryId: int、tagId: int、page: int（default 1）、pageSize: int（default 20） |
| レスポンス（200 OK） | `{ "total": 100, "page": 1, "pageSize": 20, "items": [{ "id": 1, "title": "...", "categoryName": "...", "tags": [...], "updatedAt": "..." }] }` |
| レスポンス（エラー） | 400 Bad Request: パラメータ不正 / 500 Internal Server Error |
| 備考 | JWTトークンなし→公開中のFAQのみ返す。管理者JWTトークン付与時→全件（非公開含む）返す |

#### A-003: FAQ新規登録

| 項目 | 内容 |
|---|---|
| エンドポイント | POST /api/v1/faqs |
| リクエストボディ | `{ "title": "string（必須・最大100）", "body": "string（必須）", "categoryId": "int（必須）", "tagIds": [int], "isPublished": boolean }` |
| レスポンス（201 Created） | `{ "id": 1, "title": "...", ... }` |
| レスポンス（エラー） | 400 Bad Request: バリデーションエラー / 401 Unauthorized / 500 Internal Server Error |
| 備考 | タグIDは既存タグのみ指定可能。存在しないIDは400を返す |

## 6. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | FAQ一覧・検索のレスポンスタイム2秒以内（同時接続10ユーザー程度を想定） |
| セキュリティ | JWT認証を実装（一般利用者・管理者の2ロール）。HTTPS通信を必須とする。フェーズ2で権限管理・二要素認証を追加予定 |
| 可用性 | 開発・検証用のため高可用性要件は設けない。バックアップはAzureの標準機能を利用 |
| 保守性 | 構造化ログ（JSON）を出力。エラーはスタックトレースを含めて記録する |
| 拡張性 | フェーズ2のAI機能追加に備え、APIはREST設計とし変更を最小化できる構造とする |
| アクセシビリティ | 社内ツールのため必須要件としないが、基本的なキーボード操作を可能とする |

## 8. テスト方針

| テスト種別 | 方針 |
|---|---|
| 単体テスト | バックエンドのサービスクラス・バリデーションロジックをxUnitで検証。カバレッジ目標70%以上 |
| 結合テスト | APIエンドポイント単位でのリクエスト/レスポンスを検証（TestContainers等を活用） |
